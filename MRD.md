# Market Requirements Document (MRD)

## 미국 이주 지원 플랫폼 v1.1

---

## 1. 시장 개요

### 1.1 시장 정의

미국 이주를 준비하는 의사, 회사 직원, 학생과 이를 지원하는 한인 에이전트를 위한 전문화된 정보 관리 및 커뮤니케이션 플랫폼 시장

### 1.2 타겟 시장 규모

- **초기 타겟**: LA/OC 지역 한인 에이전트 1-3명
- **연간 예상 클라이언트**: 5-10명
- **시장 특성**: 소규모 집중 시작, 니치 마켓
- **확장 가능성**: 성공 시 다른 지역(뉴욕, 시카고 등)으로 확장

### 1.3 시장 성장 동력

- 미국 이주 수요 지속 증가 (의사, IT 직원, 학생)
- 디지털화 추세 (Excel/카카오톡 → 통합 플랫폼)
- 클라이언트 셀프서비스 니즈 증가
- 에이전트 효율성 향상 필요

## 2. 경쟁 분석

### 2.1 직접 경쟁사

**현재 없음**

- 한인 이주 특화 플랫폼은 시장에 존재하지 않음
- 일반 CRM (HubSpot, Salesforce)는 이주 프로세스 특화 기능 부재

### 2.2 간접 경쟁사

#### A. Excel + 카카오톡 (현재 방식)

**강점**:

- 무료
- 익숙한 도구
- 즉시 사용 가능

**약점**:

- 정보 분산 관리
- 버전 관리 어려움
- 실시간 협업 불가
- 클라이언트 직접 접근 불가

#### B. 일반 부동산 CRM

**강점**:

- 부동산 거래 특화 기능
- 시장 검증된 도구

**약점**:

- 이주 프로세스 미지원
- 한국어 UI 부재
- 체크리스트/가족정보 기능 없음
- 한인 에이전트 특화 기능 부재

### 2.3 경쟁 우위 (차별화 포인트)

1. **이주 프로세스 전문화**

   - 단계별 체크리스트 (출국 전 → 입국 직후 → 정착)
   - 가족정보/비상연락망 구조화
   - 이주 형태별 맞춤 정보 관리

2. **한국어 UI + 실시간 커뮤니케이션**

   - 완전 한국어 인터페이스
   - 내장 채팅 시스템 (카카오톡 대체)
   - 에이전트-클라이언트 실시간 소통

3. **클라이언트 셀프서비스**

   - 클라이언트가 직접 정보 입력/수정
   - 진행상황 시각화
   - 에이전트 부담 감소

4. **올인원 플랫폼**
   - 정보 관리 + 채팅 + 체크리스트 통합
   - 단일 플랫폼에서 모든 프로세스 관리

## 3. 타겟 고객

### 3.1 Primary Persona: 에이전트 (Agent)

**인구통계**:

- 위치: LA/OC 지역
- 연령: 30-50대
- 경력: 미국 이주 지원 3년 이상

**행동 패턴**:

- 현재 Excel + 카카오톡으로 클라이언트 관리
- 클라이언트당 평균 3-6개월 지원 기간
- 동시에 2-5명 클라이언트 관리

**Pain Points**:

- 여러 클라이언트 정보 추적 어려움
- 클라이언트별 진행상황 파악 어려움
- 정보 요청 시 반복적인 질문/답변
- 카카오톡 대화 기록 관리 어려움

**Needs**:

- 클라이언트 정보 중앙 집중 관리
- 진행상황 한눈에 파악
- 효율적인 커뮤니케이션
- 클라이언트 셀프서비스로 업무 부담 감소

**구매 결정 요인**:

- 사용 편의성 (학습 곡선 낮음)
- 시간 절약 효과
- 클라이언트 만족도 향상
- **무료 사용** (에이전트는 비용 부담 없음)

### 3.2 Secondary Persona: 클라이언트 (Client)

**인구통계**:

- 직업: 의사, 회사 직원, 학생
- 연령: 25-45세
- 위치: 한국 (이주 준비 중)

**행동 패턴**:

- 이주 3-6개월 전부터 준비 시작
- 에이전트와 주 1-2회 소통
- 정보 입력/수정 빈번

**Pain Points**:

- 자신의 정보를 직접 관리할 수 없음
- 진행상황 파악 어려움
- 에이전트에게 반복 질문 필요
- 여러 채널에 정보 분산

**Needs**:

- 본인 정보 직접 입력/수정
- 준비사항 체크리스트 확인
- 진행상황 시각화
- 에이전트와 실시간 소통

**구매 결정 요인**:

- 사용 편의성
- 정보 투명성
- 실시간 소통 가능

## 4. 시장 기회

### 4.1 시장 진입 전략

**Phase 1: 소규모 집중 (0-6개월)**

- LA/OC 지역 1-3명 에이전트 타겟
- 직접 영업 및 온보딩 지원
- 피드백 수집 및 제품 개선

**Phase 2: 지역 확장 (6-12개월)**

- 뉴욕, 시카고 등 다른 지역 확장
- 입소문 마케팅
- 에이전트 추천 프로그램

**Phase 3: 기능 확장 (12개월+)**

- 파일 공유, 캘린더 통합 등 고급 기능
- 모바일 앱 (PWA)
- 다국어 지원 (영어)

### 4.2 수익 모델

#### A. 소프트웨어 서비스 + 정보 관리

- **가격**: $300 (1회)
- **포함 내용**:
  - 플랫폼 무제한 사용 (이주 완료까지)
  - 체크리스트 + 채팅 + 정보 관리
- **결제 시점**: 클라이언트 등록 시
- **결제 주체**: 클라이언트 (에이전트는 무료 사용)

#### B. 중개 수수료

- **가격**: 월세의 100% (임대 계약 시)
- **예시**: 월세 $3,000 → 수수료 $3,000
- **결제 시점**: 임대 계약 체결 시
- **대상**: 에이전트

### 4.3 시장 성장 전망

- **Year 1**: 1-3명 에이전트, 5-10명 클라이언트
- **Year 2**: 5-10명 에이전트, 20-30명 클라이언트
- **Year 3**: 10-20명 에이전트, 50-100명 클라이언트

## 5. 시장 리스크

### 5.1 시장 리스크

1. **소규모 시장**

   - 타겟 고객 수가 제한적
   - 완화: 니치 마켓 집중, 높은 고객 충성도 목표

2. **에이전트 온보딩 저항**

   - 기존 방식(Excel/카카오톡)에 익숙함
   - 완화: 직접 온보딩 지원, 명확한 ROI 제시

3. **클라이언트 정보 입력 동기 부족**
   - 수동 입력 회피
   - 완화: 진행상황 시각화, 필수/선택 필드 구분

### 5.2 경쟁 리스크

1. **대형 CRM 플랫폼의 이주 특화 기능 추가**

   - 완화: 빠른 시장 진입, 고객 로열티 구축

2. **유사 서비스 출현**
   - 완화: 첫 무버 어드밴티지, 지속적인 기능 개선

### 5.3 기술 리스크

1. **플랫폼 안정성**

   - 완화: Vercel + Supabase 검증된 인프라

2. **데이터 보안**
   - 완화: Clerk + Supabase RLS, GDPR/CCPA 준수

## 6. 성공 지표 (KPI)

### 6.1 시장 진입 지표

- **에이전트 가입 수**: Year 1 목표 3명
- **클라이언트 등록 수**: Year 1 목표 10명
- **에이전트당 평균 클라이언트 수**: 목표 3-4명

### 6.2 참여 지표

- **클라이언트 활성 사용자**: 주간 로그인 70% 이상
- **체크리스트 완료율**: 평균 80% 이상
- **평균 채팅 메시지 수**: 클라이언트당 20건 이상
- **프로필 완성도**: 필수 항목 100% 입력

### 6.3 비즈니스 지표

- **수익**: Year 1 목표 $3,000 (서비스, 클라이언트 결제) + $30,000 (수수료)
- **고객 만족도**: NPS 50 이상
- **에이전트 재사용률**: 100% (무료 사용으로 지속적 활용)

## 7. 마케팅 전략

### 7.1 초기 마케팅 (Phase 1)

- **직접 영업**: LA/OC 지역 에이전트 직접 접촉
- **온보딩 지원**: 1:1 온보딩 세션 제공
- **피드백 수집**: 초기 사용자와 긴밀한 소통

### 7.2 성장 마케팅 (Phase 2+)

- **입소문 마케팅**: 만족한 에이전트 추천
- **콘텐츠 마케팅**: 이주 가이드 블로그
- **에이전트 추천 프로그램**: 추천 시 인센티브

---

## 8. 결론

미국 이주 지원 플랫폼은 **니치하지만 명확한 시장**을 가지고 있으며, **직접 경쟁사가 없는 블루오션**입니다. 소규모 집중 전략으로 시작하여 점진적으로 확장하는 것이 성공의 핵심입니다.

**핵심 성공 요인**:

1. 에이전트의 실제 Pain Point 해결
2. 클라이언트 셀프서비스로 에이전트 부담 감소
3. 사용 편의성 (학습 곡선 낮음)
4. 빠른 시장 진입 및 고객 로열티 구축
