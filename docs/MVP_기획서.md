# MVP 기획서

## 미국 이주 지원 플랫폼 v1.1

---

## 1. 서비스 개요

### 1.1 서비스명

**미국 이주 지원 플랫폼** (Lala Landing)

### 1.2 서비스 정의

미국 이주를 준비하는 의사, 회사 직원, 학생과 이를 지원하는 한인 에이전트를 위한 전문화된 정보 관리 및 커뮤니케이션 플랫폼

### 1.3 핵심 가치 제안

1. **정보 수집 자동화**: 구조화된 프로필 및 요구사항 입력
2. **준비 절차 가시화**: 단계별 체크리스트 제공
3. **실시간 커뮤니케이션**: 내장 채팅 시스템 (카카오톡 대체)
4. **효율적 관리**: 에이전트 대시보드를 통한 다중 클라이언트 관리
5. **클라이언트 셀프서비스**: 클라이언트가 직접 정보 입력/수정 가능

### 1.4 차별화 포인트

- **이주 프로세스 전문화**: 단계별 체크리스트, 가족정보/비상연락망 구조화
- **한국어 UI**: 완전 한국어 인터페이스
- **올인원 플랫폼**: 정보 관리 + 채팅 + 체크리스트 통합
- **직접 경쟁사 없음**: 한인 이주 특화 플랫폼은 시장에 존재하지 않음

---

## 2. 목적

### 2.1 비즈니스 목표

- **에이전트 효율성 향상**: 다중 클라이언트 관리 시간 50% 절감
- **클라이언트 만족도 향상**: 정보 투명성 및 셀프서비스 제공
- **프로세스 표준화**: 이주 준비 프로세스 체계화
- **시장 진입**: LA/OC 지역 1-3명 에이전트, 연간 5-10명 클라이언트 타겟

### 2.2 사용자 목표

#### 에이전트

- 여러 클라이언트를 한 곳에서 효율적으로 관리
- 클라이언트별 진행상황 실시간 파악
- 정보 요청 시 반복적인 질문/답변 최소화
- 카카오톡 대화 기록 관리 어려움 해소

#### 클라이언트

- 본인 정보를 직접 입력/수정하여 투명성 확보
- 준비사항 체크리스트로 진행상황 명확히 파악
- 에이전트와 실시간 소통으로 빠른 피드백

## 3. 배경

### 3.1 문제 정의

#### 현재 상황

- **정보 분산**: 이주 관련 정보가 Excel, 카카오톡, 이메일 등 여러 채널에 분산
- **관리 어려움**: 에이전트가 클라이언트별 진행상황을 추적하기 어려움
- **비효율성**: 클라이언트가 자신의 정보를 직접 관리할 수 없어 에이전트에게 반복 질문
- **커뮤니케이션 한계**: 에이전트-클라이언트 간 실시간 소통 채널 부재

#### 시장 환경

- 미국 이주 수요 지속 증가 (의사, IT 직원, 학생)
- 디지털화 추세 (Excel/카카오톡 → 통합 플랫폼)
- 클라이언트 셀프서비스 니즈 증가
- 에이전트 효율성 향상 필요

### 3.2 시장 기회

- **직접 경쟁사 없음**: 한인 이주 특화 플랫폼은 시장에 존재하지 않음
- **니치 마켓**: 소규모 집중 시작으로 빠른 시장 진입 가능
- **블루오션**: 일반 CRM은 이주 프로세스 특화 기능 부재

### 3.3 수익 모델

#### A. 소프트웨어 서비스 + 정보 관리

- **가격**: $300 (1회)
- **결제 주체**: 클라이언트
- **포함 내용**: 플랫폼 무제한 사용 (이주 완료까지), 체크리스트 + 채팅 + 정보 관리
- **에이전트**: 무료 사용

#### B. 중개 수수료

- **가격**: 월세의 100% (임대 계약 시)
- **예시**: 월세 $3,000 → 수수료 $3,000
- **대상**: 에이전트

## 4. 타깃 유저

### 4.1 Primary User: 에이전트 (Agent)

#### 인구통계

- **위치**: LA/OC 지역
- **연령**: 30-50대
- **경력**: 미국 이주 지원 3년 이상
- **현재 도구**: Excel + 카카오톡으로 클라이언트 관리

#### 행동 패턴

- 클라이언트당 평균 3-6개월 지원 기간
- 동시에 2-5명 클라이언트 관리
- 에이전트와 주 1-2회 소통

#### Pain Points

- 여러 클라이언트 정보 추적 어려움
- 클라이언트별 진행상황 파악 어려움
- 정보 요청 시 반복적인 질문/답변
- 카카오톡 대화 기록 관리 어려움

#### Needs

- 클라이언트 정보 중앙 집중 관리
- 진행상황 한눈에 파악
- 효율적인 커뮤니케이션
- 클라이언트 셀프서비스로 업무 부담 감소

#### 구매 결정 요인

- 사용 편의성 (학습 곡선 낮음)
- 시간 절약 효과
- 클라이언트 만족도 향상
- **무료 사용** (에이전트는 비용 부담 없음)

### 4.2 Secondary User: 클라이언트 (Client)

#### 인구통계

- **직업**: 의사, 회사 직원, 학생
- **연령**: 25-45세
- **위치**: 한국 (이주 준비 중)

#### 행동 패턴

- 이주 3-6개월 전부터 준비 시작
- 에이전트와 주 1-2회 소통
- 정보 입력/수정 빈번

#### Pain Points

- 자신의 정보를 직접 관리할 수 없음
- 진행상황 파악 어려움
- 에이전트에게 반복 질문 필요
- 여러 채널에 정보 분산

#### Needs

- 본인 정보 직접 입력/수정
- 준비사항 체크리스트 확인
- 진행상황 시각화
- 에이전트와 실시간 소통

#### 구매 결정 요인

- 사용 편의성
- 정보 투명성
- 실시간 소통 가능
- $300 (1회) 가격 대비 가치

## 5. 핵심 기술

### 5.1 기술 스택

| 카테고리           | 기술                  | 버전              | 용도                       |
| ------------------ | --------------------- | ----------------- | -------------------------- |
| **Frontend**       | Next.js               | 15.x (App Router) | 웹 애플리케이션 프레임워크 |
| **Language**       | TypeScript            | 5.x               | 타입 안정성                |
| **Styling**        | Tailwind CSS          | 3.x               | 스타일링                   |
| **Authentication** | Clerk                 | Latest            | 사용자 인증 및 권한 관리   |
| **Database**       | Supabase (PostgreSQL) | Latest            | 데이터 저장 및 관리        |
| **Real-time**      | Supabase Realtime     | Latest            | 실시간 채팅 (Phase 2)      |
| **Hosting**        | Vercel                | Hobby Plan        | 프론트엔드 배포            |
| **Backend**        | Next.js API Routes    | -                 | 서버 사이드 로직           |

### 5.2 시스템 아키텍처

```
┌─────────────────┐
│   User Browser  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Vercel Edge   │
│   (Next.js App) │
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
┌────────┐ ┌──────────┐
│ Clerk  │ │ Supabase │
│  Auth  │ │   DB     │
└────────┘ └──────────┘
```

### 5.3 핵심 기술 특징

#### 인증 시스템

- **Clerk 기반**: 이메일/소셜 로그인 지원 (Google, 카카오 등)
- **역할 기반 접근 제어**: 에이전트/클라이언트 구분
- **자동 라우팅**: 역할에 따라 자동으로 대시보드/홈으로 이동

#### 데이터베이스

- **Supabase (PostgreSQL)**: 관계형 데이터베이스
- **Row Level Security (RLS)**: 데이터 접근 제어
- **자동 백업**: 일 1회 자동 백업

#### 실시간 통신

- **Phase 1**: 폴링 방식 (5초 간격)
- **Phase 2**: Supabase Realtime (WebSocket 기반 즉시 전송)

#### 보안

- **HTTPS 필수**: 모든 통신 암호화
- **RLS 정책**: 사용자별 데이터 접근 제어
- **GDPR/CCPA 준수**: 개인정보 보호 규정 준수

## 6. UX / 서비스 흐름

### 6.1 페이지 구조 (Sitemap)

```
/ (랜딩 페이지)
├── 서비스 소개
├── [에이전트 로그인] 버튼 → Clerk SignIn
└── [클라이언트 로그인] 버튼 → Clerk SignIn

/sign-in (Clerk 제공)
└── 통합 로그인 페이지 (역할 자동 판별)

/sign-up (Clerk 제공)
└── 회원가입 + 역할 선택

/agent/dashboard
├── 클라이언트 목록 (카드/테이블)
├── [+ 새 클라이언트 추가] 버튼
└── 각 클라이언트 카드 클릭 → /agent/client/[id]

/agent/client/[id]
├── 탭 네비게이션:
│   ├── 프로필
│   ├── 주거옵션
│   ├── 체크리스트
│   └── 채팅
└── 우측/하단: 채팅창 고정 (선택적)

/client/home
├── 탭 네비게이션:
│   ├── 내 프로필
│   ├── 주거옵션
│   ├── 체크리스트
│   └── 채팅
└── 우측/하단: 채팅창 고정 (선택적)
```

### 6.2 디자인 원칙

#### 언어

- **전체 UI**: 한국어
- 메뉴, 버튼, 레이블 모두 한글 사용

#### 디자인 철학

- **단순함**: 정보 과부하 방지, 한 화면에 한 가지 핵심 기능
- **직관성**: 에이전트/클라이언트 모두 별도 교육 없이 사용 가능
- **반응형**: 모바일 대응 (특히 채팅 기능)
- **시각적 피드백**: 로딩 상태, 성공/에러 메시지 명확히 표시

#### 컬러 스킴

- **Primary**: 블루 계열 (신뢰감)
- **Success**: 그린 (체크리스트 완료)
- **Warning**: 옐로우 (주의사항)
- **Error**: 레드 (오류)

#### 주요 컴포넌트

- **카드 레이아웃**: 클라이언트 목록, 프로필 요약
- **탭 네비게이션**: 클라이언트 상세 페이지
- **프로그레스 바**: 체크리스트 완료율
- **채팅 UI**: 카카오톡 스타일 말풍선
- **폼**: 명확한 레이블, 인라인 유효성 검사

### 6.3 주요 화면 흐름

#### 에이전트 플로우

1. **랜딩 페이지** → 에이전트 로그인 버튼 클릭
2. **로그인/회원가입** → 역할 선택 (에이전트)
3. **대시보드** → 클라이언트 목록 확인
4. **클라이언트 추가** → 새 클라이언트 정보 입력
5. **클라이언트 상세** → 프로필/주거옵션/체크리스트/채팅 관리

#### 클라이언트 플로우

1. **랜딩 페이지** → 클라이언트 로그인 버튼 클릭
2. **로그인/회원가입** → 역할 선택 (클라이언트)
3. **홈 페이지** → 내 프로필/주거옵션/체크리스트/채팅 확인
4. **정보 입력/수정** → 직접 프로필 및 주거 요구조건 입력
5. **체크리스트 확인** → 진행상황 시각화 및 완료 표시
6. **채팅** → 에이전트와 실시간 소통

### 7.3 주요 상호작용 시나리오

#### 시나리오 1: 에이전트가 새 클라이언트 추가

1. 대시보드에서 "+ 새 클라이언트 추가" 버튼 클릭
2. 클라이언트 기본 정보 입력 (이름, 이메일, 직업, 이주 예정일)
3. 저장 후 클라이언트 상세 페이지로 이동
4. 주거 요구사항 입력 안내
5. 체크리스트 자동 생성 확인

#### 시나리오 2: 클라이언트가 정보 입력

1. 로그인 후 홈 페이지 접속
2. "내 프로필" 탭에서 기본 정보 입력
3. "주거옵션" 탭에서 주거 요구조건 입력
4. "체크리스트" 탭에서 진행상황 확인
5. "채팅" 탭에서 에이전트에게 질문

#### 시나리오 3: 에이전트-클라이언트 채팅

1. 클라이언트가 채팅으로 질문 전송
2. 에이전트가 대시보드에서 새 메시지 알림 확인
3. 클라이언트 상세 페이지의 채팅 탭으로 이동
4. 메시지 확인 후 답변 전송
5. 클라이언트가 실시간으로 답변 확인 (폴링 방식)

### 8.5 입국 후 체크리스트 (P0 - 필수)

**기능**: 단계별 준비사항 자동 생성 및 관리

**체크리스트 카테고리** (템플릿 고정):

#### 1. 출국 전 준비 (7일 전)

- **국제운전면허증 발급**

  - 한국에서 국제운전면허증 발급
  - 거주자 판단 기준: 캘리포니아에서의 체류 의도, 주소 확보, 장기 체류 증거
  - 거주자의 경우 인정 기간: 10일
  - 여행자 신분으로 본국의 운전면허증과 반드시 같이 지참 필요

- **비자 확인**

  - 비자 유효기간 확인
  - 입국 관련 서류 준비

- **항공권 예약**

  - 입국 일정 확정
  - 항공권 예약 및 확인

- **짐 정리**
  - 필수 물품 준비
  - 이주 짐 정리

#### 2. 입국 직후 (1주차)

- **집 렌트** (가장 먼저 진행)

  - 집 종류 선택: 아파트(1~2 bedroom), 타운하우스(3~4 bedroom), 하우스(3~4 bedroom)
  - ZILLOW로 한국에서 사전 시세 확인
  - 계약 시 필요 서류 준비:
    - SSN (없으면 보증인 필요 또는 보증금 과다)
    - 급여명세서 (Paystub)
    - 은행 잔고증명 (Bank Statement)
  - 보증금(Deposit) + 첫 달 렌트 선납 준비
  - Background 체크 기간 확인 (2~3주 소요 가능)
  - 조건 확인: 게이트(보안), 세탁기/건조기 구비, 마루 바닥, 회사 근접 거리

- **SSN 발급 신청**

  - 신청 장소: 가까운 Social Security Office
  - 필요 서류: 여권, 비자, I-94(입국기록), SS-5 양식
  - SS-5 양식 인터넷 다운로드 후 작성
  - 기간: 신청 후 2~3주 소요 (우편 배달)
  - **최대한 빨리 신청 필요** (집 렌트, 은행 계좌 개설 등에 필요)

- **은행 계좌 개설**

  - 즉시 신청 가능: 신한은행(미국 지사)
  - 주요 미국 은행: Chase, Bank of America, Wells Fargo 등
  - 필요 서류: 여권, SSN, 거주지 증명(집 렌트 계약서 등)
  - 계좌 종류: Saving Account, Checking Account 2개 개설
  - 최소 잔고: 약 $2,000 (Chase 기준, 이용료 면제)
  - 초기: Checking Account + Debit Card 개설
  - 향후: 신용카드 발급 (Credit 쌓기 시작, Credit limit 약 $1,000)

- **유틸리티 신청**
  - 집 입주 날짜 정해지면 신청
  - 필요 서류: SSN, 집 렌트 계약서
  - 신청 항목:
    - **수도**: 거주하는 시의 수도 업체에 온라인/전화 신청
    - **전기**: SoCal Edison 온라인/전화 신청
    - **가스**: SoCal Gas 온라인/전화 신청
    - **인터넷**: Spectrum, AT&T, Verizon 등 업체 온라인/전화 신청
    - **쓰레기 수거**: EDCO 등 거주하는 시의 수거 업체에 신청
  - 참고: 일부 아파트는 월세 납부 시 유틸리티 함께 청구
  - 유틸리티 청구서는 은행 개설 및 DMV 시 거주증명으로 활용

#### 3. 정착 단계 (1개월차)

- **운전면허 취득**

  - DMV 온라인 계정 생성 및 정보 기입 후 방문 추천
  - 필기 시험: 온라인 또는 DMV 방문
    - 한국어 선택 가능
    - 예상 문제: 라디오코리아, 유튜브 참고 (30분 공부 추천)
    - 난이도가 높아져 유튜브 공부 필요
  - 필요 서류: 여권, SSN, 거주지 증명서류(집 렌트 계약서, 유틸리티 영수증 등 2개 이상), I-94
  - 필기 시험 통과 시 Learner's Permit (임시 면허) 발급
  - 임시 면허 발급 후: 운전면허 소지자와 동승하여 운전 가능
  - 실기 시험: DMV 홈페이지에서 예약, 자차로 시험 응시
    - 지역별 난이도 상이 → 해당 지역 실기 시험 유튜브 시청 및 연습 필요
  - **최대한 빠른 취득 및 안전운전 필수**

- **차량 구매/리스**

  - 미국은 개개인마다 차 필요
  - 신차: 각 자동차 매매업체 or 중개인을 통해 구매
  - 중고차: Carmax 방문하여 중고차 구매
  - 필요 서류: 여권 사본, 비자 사본, 운전면허증 사본, I-94, Job Offer
  - 참고: 시기별 프로모션 확인, 다양한 옵션 고려

- **자동차 보험**

  - 차량 구매/리스와 동시에 진행
  - 보험사별 비교 후 선택
  - 운전기록 없으면 견적 거부 가능
  - 초기 보험료: 월 약 $350

- **자녀 학교 등록** (해당 시)

  - 거주지 학군 확인
  - 학교 등록 서류 준비
  - 등록 완료

- **의료보험 가입**
  - 회사 제공 보험 확인
  - 개인 보험 가입 (필요 시)
  - 보험 카드 수령

**기능**:

- 체크박스 토글
- 완료율 시각화 (프로그레스 바)
- 각 항목별 상세 설명 및 필요 서류 표시
- 에이전트/클라이언트 모두 업데이트 가능
- 항목별 중요도 표시 (예: 집 렌트 - 가장 먼저, SSN - 최대한 빨리)

**Phase 2 추가 예정**:

- 각 항목별 메모 추가 기능
- 커스터마이징 기능 (에이전트가 항목 추가/삭제)
- 항목별 체크리스트 하위 항목 추가 (예: 유틸리티 → 수도, 전기, 가스, 인터넷, 쓰레기 수거)

- 직접 온보딩 지원 및 명확한 ROI 제시
- 진행상황 시각화로 동기 부여
- 필수/선택 필드 구분으로 UX 부담 완화
- 무료 플랜으로 시작하여 필요 시 확장
